# プロダクトレッドグロース プレイブック — フリーミアムからエンタープライズへの転換

プロダクトに営業させる。このプレイブックはPLGモーション全体をカバー。

## フリーミアム設計

何をゲートし、何を無料で提供し、どう価格心理を活用するか。

- **バリューメトリック**: 顧客が自然に数える単位を特定（シート、プロジェクト、ストレージ、APIコール）
- **無料枠の範囲**: 「アハモーメント」に到達するには十分だが、ビジネスを回すには不十分な範囲
- **アップグレードトリガー**: 実際の価値を得ていることと相関する利用制限（恣意的な壁ではなく）
- **価格心理**: 高くアンカリング、節約を見せ、ミドルティアを明らかな選択肢にする

## セルフサーブオンボーディング

興味を失う前にユーザーをアクティベートするため、タイムトゥバリューを短縮。

- **初回セッション目標**: 「アクティベート済み」の定義（最初のXを作成、チームメイトを招待、セットアップ完了）
- **プログレッシブディスクロージャー**: 一度にすべてを見せない — キーアクションを順番にガイド
- **空の状態**: すべてのブランク画面は教育と行動促進の機会
- **マイルストーン祝福**: コミットメントを強化するために進捗を視覚的に表示

## バイラルメカニクス

ビルトインのシェアリング、コラボレーションフック、ネットワークエフェクト。

- **コラボレーション by デザイン**: 複数ユーザーが必要なプロダクトはより速く広がる
- **シェア可能な成果物**: 自然に共有されるアウトプット（レポート、リンク、埋め込み）
- **リファラルプログラム**: 紹介者と被紹介者の両方にプレミアム機能を報酬として提供
- **ネットワークエフェクト**: 新規ユーザーが増えるたびに既存ユーザーへの価値も向上

## 使用量ベースシグナル

エクスパンション可能なアカウントを自動的に特定。

- **PQL定義**: Product Qualified Lead＝使用量閾値を一貫して超えるアカウント
- **シグナルスコアリング**: インテントレベルでアクションに重み付け（チームメイト招待 > ドキュメント編集 > ダッシュボード閲覧）
- **自動アラート**: アカウントがPQL閾値を超えた時にセールスに通知
- **タイミング**: トライアル期限切れ時ではなく、価値が実証された時にアプローチ

## エンタープライズアップセル

いつ、どのようにPLGモーションにセールスを重ねるか。

- **ボトムアップ採用**: 個人ユーザー→チーム採用→部門→エンタープライズ契約
- **エンタープライズトリガー**: 5人以上のアクティブシート、コンプライアンス要件、SSO要求、API使用量急増
- **セールスアシストモデル**: セールスはゲートキーピングではなく価値を追加（カスタムオンボーディング、セキュリティレビュー）
- **契約拡張**: チームで入り、成功事例を通じて組織全体に拡大
